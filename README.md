# evhedge

EV-анализ и планировщик хеджей для long-shot ставок на победу в турнире
(например, покупка "NO"-акций на Polymarket на то, что команда/игрок **не**
выиграет турнир целиком) с постадийным хеджированием на классических
букмекерских линиях (Pinnacle и т.п.) по мере прохождения командой раундов.

## Статус

Реализованы: модели данных (`evhedge/models.py`), точный (без Monte Carlo)
EV-движок (`evhedge/engine.py`) и планировщик размера хедж-ставок
(`evhedge/strategies.py`), покрытые тестами на руками посчитанных примерах.
Остальные модули (`montecarlo`, `ranking`, `config_io`, `cli`, `sports/*`,
`data_sources/*`) — заготовки под дальнейшую реализацию.

## Установка

```bash
pip install -e .
```

## Идея

1. Турнир для команды представлен как `Bracket` — упорядоченный список
   `Stage` (раундов), каждый со своей условной вероятностью прохождения
   (`win_prob`) и, опционально, коэффициентом для хеджа на этом раунде
   (`hedge_decimal_odds`).
2. `MarketPrices` описывает первичную позицию — покупку NO-акций на
   Polymarket (цена, размер ставки, банкролл).
3. `StrategyConfig` задаёт, как и когда докупать хедж по ходу турнира
   (на каждой стадии, только в финале, по Kelly, с фиксацией прибыли и т.д.).
4. `evhedge.engine.compute_ev` строит таблицу исходов (`OutcomeRow`) —
   команда вылетает на стадии *k* или выигрывает турнир — и агрегирует её
   в `EVResult` (ожидаемая прибыль, EV на доллар риска, дисперсия).
5. `evhedge.montecarlo` симулирует то же самое по путям, чтобы оценить VaR
   и просадку по банкроллу.
